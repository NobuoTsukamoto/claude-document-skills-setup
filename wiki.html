docx スキル 環境セットアップ

Claude Code の docx / pptx スキルを Windows で動かすための依存環境。Word文書の作成・読取・編集・PDF化・画像化、およびスライドの作成・読取・画像化まで動く状態にします。

Platform Windows 10 / 11 Shell PowerShell 7+ 検証済 日本語環境 冪等 再実行OK

TL;DR 自動セットアップ

前提として winget / uv / node(npm) が導入済みであること。無ければ下記を先に。冪等なスクリプトなので、何度実行しても安全です。

PowerShell · 前提（未導入なら）
winget install astral-sh.uv
winget install OpenJS.NodeJS
PowerShell · 実行（このフォルダで）
pwsh -File .\setup-document-skills.ps1

最後に [SUCCESS] が出れば完了。新しいターミナルを開き直すと PATH 等が反映されます。確認だけしたいときは -VerifyOnly を付けて実行します。

DEPLOY チームへの展開

このスクリプトが用意するのは 依存環境だけ。docx スキル本体は Claude Code のプラグインとして別途入っている必要があります。次の順で案内してください。

前提ツール（未導入なら）

winget から uv と Node.js を入れる。

PowerShell
winget install astral-sh.uv
winget install OpenJS.NodeJS

docx / pptx スキル（プラグイン）を導入

Claude Code 内で /plugin から anthropic-agent-skills の document-skills（docx / pptx を含む）を有効化する。これが無いと scripts/office/*.py のパスが存在せず、セットアップだけでは動きません。

リポジトリを clone して実行

PowerShell
git clone https://github.com/NobuoTsukamoto/claude-document-skills-setup.git
cd claude-document-skills-setup
pwsh -File .\setup-document-skills.ps1
エージェント向けガイド。 Claude がスキルを正しく使うための実行時ルール（既定の python ではなく venv の python を使う 等）は AGENTS.md に置いています。マシン全体に効かせるには、その内容を各自の ~/.claude/CLAUDE.md に取り込んでください。リポジトリ内で作業する場合は同梱 CLAUDE.md（@AGENTS.md を取り込み）が自動で読まれます。

03 ハマりどころ

今回の構築で実際に踏んだ落とし穴。チーム展開時の参考に。

Pitfall 01 · Shell

Claude Code の ! は PowerShell ではなく bash で動く

! プレフィックスのコマンドは Git Bash で実行されます。PowerShell 構文（[Environment]::…）を ! で貼ると構文エラー。環境変数の永続設定は .ps1 スクリプト か PowerShell を直接起動して行う。

Pitfall 02 · Env

環境変数の変更は「後から起動したプロセス」にしか効かない

winget や SetEnvironmentVariable は Windows 側を更新するが、既に動いているターミナル / Claude Code は起動時の古いコピーを持ち続ける。反映にはターミナルを閉じて開き直す（PC 再起動までは不要）。

Pitfall 03 · PATH

winget の「コマンドラインエイリアス」が作られないことがある

Windows の開発者モードが無効だと WinGet\Links のシンボリックリンク作成が静かに失敗し、pandoc 等が PATH に出ない。本スクリプトは実体フォルダを直接 PATH に追加するため開発者モード不要。

Pitfall 04 · Versioning

pandoc / poppler はバージョン番号入りフォルダ

例: …\pandoc-3.10\、…\poppler-25.07.0\。winget upgrade で版数が変わり PATH が切れる。スクリプトを再実行すれば新フォルダを解決して自己修復する（LibreOffice / NODE_PATH / PYTHONUTF8 は影響なし）。

Pitfall 05 · Encoding

日本語Windowsの cp932 エラー

PYTHONUTF8=1 未設定だと validate.py 等が UTF-8/日本語XMLの読取で 'cp932' codec can't decode … を出す。この変数の設定で解消。

Pitfall 06 · Python

スキルスクリプトは venv の python で実行する

SKILL.md には python scripts/… とあるが、Windows 既定の python は Microsoft Store のスタブ（実体なし）のことが多い。スキルの Python スクリプトは …\.claude\skill-envs\docx\Scripts\python.exe で実行する。

Pitfall 07 · LibreOffice

同梱の soffice.py は Windows 非対応 → soffice 直呼び

scripts/office/soffice.py は socket.AF_UNIX を使う Unix 専用ラッパーで、Windows では AttributeError で失敗する。PDF変換・画像化は soffice --headless --convert-to pdf … を直接呼ぶ（pptx の画像プレビューも同様）。

Pitfall 08 · pptxgenjs

色の # の扱いが逆：pptxgenjs は無し / react-icons は必須

pptxgenjs のhex色は "2DD4BF"（# を付けると破損）。一方 react-icons に渡す色は #2DD4BF と # 必須（無いと黒にフォールバックし低コントラスト化）。日本語は fontFace に日本語フォント（例 Yu Gothic UI）を指定する。

04 動作確認（エンドツーエンド）

PowerShell
$py    = "$env:USERPROFILE\.claude\skill-envs\docx\Scripts\python.exe"
$skill = "$env:USERPROFILE\.claude\plugins\marketplaces\anthropic-agent-skills\skills\docx"

# 生成 (docx-js)
node -e "const {Document,Packer,Paragraph,TextRun}=require('docx');const fs=require('fs');Packer.toBuffer(new Document({sections:[{children:[new Paragraph({children:[new TextRun('テスト 環境確認')]})]}]})).then(b=>fs.writeFileSync('test.docx',b))"

pandoc test.docx -t plain                       # 読取
& $py "$skill\scripts\office\unpack.py"   test.docx unpacked   # 展開
& $py "$skill\scripts\office\validate.py" test.docx           # 検証

✓ All validations PASSED! が出れば環境構築は完成です。
